# Task Prompt

Construct a Rural Land Lease & Settlement Management Portal supporting offline administration of agricultural lease contracts and communications across a county. The user experience is delivered as a Layui-based web interface with separate entry points for Farmers, Enterprises, Village Collectives, and System Admins, each seeing only the functions and geography they are authorized for (village, township, or county scope). Users register with a username and password, then complete real-name and qualification details by entering government ID numbers, business license numbers, and uploading locally stored scans; the UI clearly shows “Pending/Approved/Rejected” states and requires a rejection reason when reviewers deny verification. Profiles act as master records for people and organizations: users can create and maintain entity profiles with configurable extra fields (for example, “primary crop” or “equipment storage needs”), while the system flags likely duplicates based on matching name + address + last 4 of ID/license and offers a guided merge with side-by-side comparison and a visible change history. Lease finance is centered on contracts that auto-generate billing schedules in USD (rent, deposit, and optional property/maintenance fees), show paid/unpaid/overdue status, and calculate late charges with a default rule of 5-day grace period followed by 1.5% per month applied daily, capped at $250.00 per invoice; users can record payments, print receipts, and export ledgers and reconciliation reports to CSV/Excel for offline accounting. Secure communication is provided through an in-app conversation panel supporting text, voice notes, and images, with read indicators, message recall within 10 minutes, retention defaults of 24 months (admin-configurable), a “Report” action for harassment or fraud, and on-screen warnings when sensitive terms are detected before sending.

The backend uses ThinkPHP to implement a decoupled REST-style API consumed by the Layui frontend over the local network, with MySQL persisting accounts, roles, geographic data scope, entity master records, contracts, invoices, payments, refunds, messages, and audit trails. Role-based access control is enforced server-side on every request, including data-scope filtering so a village collective cannot view records outside its village; admin users can delegate township/county access with explicit approval. Password policy requires at least 12 characters including upper, lower, number, and symbol; lockout triggers after 5 failed attempts in 15 minutes with exponential backoff, optional TOTP-based MFA (offline authenticator app) for admins, and a local CAPTCHA to slow scripted attempts. All sensitive fields (ID/license numbers, bank references) are encrypted at rest using AES-256 with keys stored outside the web root, and displayed masked by default; all traffic is protected with local TLS when deployed on an intranet. Messaging content is encrypted in transit and at rest, with attachments validated by file type and size (max 10 MB) and stored locally with checksums to detect tampering; “risk content” detection is implemented via an offline, admin-managed keyword and pattern library with configurable actions (warn, block, or allow-and-flag). Financial reconciliation includes idempotent payment posting (duplicate submission prevention within a 10-minute window), immutable invoice snapshots, and a complete audit log of key operations (verification decisions, contract edits, bill generation, payment/refund actions, exports) including timestamp, user, IP/device fingerprint, and before/after values, enabling traceable queries without relying on any external services.
